Задание 3. Заполните таблицу dm_restaurants
Продолжите заполнением измерения «Рестораны».
Дополните ранее созданный DAG новым шагом и реализуйте логику заполнения dm_restaurants. В качестве active_from возьмите дату update_ts из MongoDB.
Как заполнять поля таблицы:
id — автогенерируемое значение, int. Пример: 1.
restaurant_id — оригинальный id ресторана из системы-источника, varchar. Пример: a51e4e31ae4602047ec52534.
restaurant_name — оригинальное имя из системы-источника, varchar. Пример: Кубдари.
active_from — update_ts из системы-источника, timestamp. Пример: 2022-10-31 18:00:14.000.
active_to — срок действия записи, timestamp. Если запись актуальна, то 2099-12-31. Пример: 2099-12-31 00:00:00.000.

В STG данные о ресторане лежат в формате JSON. Поэтому логика шага может выглядеть таким образом:
Прочитать данные из таблицы в STG.
Преобразовать JSON к объекту. Воспользуйтесь функцией str2json из файла lib/dict_util.py.
Вставить объекты с учётом историчности SCD2
