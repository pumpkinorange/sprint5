Задание 7. Заполните таблицу fct_product_sales
Таблица dds.fct_product_sales — центральная для «снежинки». Она содержит факты продаж — заказы, разбитые по позициям. В ней также учитываются оплаты и начисления бонусов, поэтому для заполнения таблицы пригодятся данные из подсистемы бонусных оплат.
Добавьте в DAG новый шаг для заполнения таблицы dds.fct_product_sales.
Как заполнять таблицу:
id — автогенерируемое значение, int. Пример: 1.
product_id — id версии продукта (FK на dm_products.id), int. Пример: 1.
order_id — id заказа (FK на dm_orders.id), int. Пример, 1.
count — количество экземпляров этого продукта в заказе, int. Пример: 3.
price — цена продукта, numeric(19, 5). Пример: 590.00000.
total_sum — общая сумма к оплате (цена * количество), numeric(19, 5). Пример: 1770.00000.
bonus_payment — сумма оплаты бонусами за этот продукт (берётся из подсистемы оплаты бонусами), numeric(19, 5). Пример: 40.00000.
bonus_grant — сумма начисленных бонусов за этот продукт (берётся из подсистемы оплаты бонусами), numeric(19, 5). Пример: 88.00000.

Подсказка:
В STG данные о транзакциях лежат в единой таблице событий. Чтобы построить таблицу фактов, понадобятся события bonus_transaction. Подход может выглядеть так:
Прочитать данные из таблицы в STG.
Преобразовать JSON к объекту.
Найти заказ по id.
Найти версию продукта по id и update_ts.
Вставить объект в таблицу фактов.
