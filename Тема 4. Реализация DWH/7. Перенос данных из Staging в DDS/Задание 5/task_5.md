Задание 5. Заполните таблицу dm_products
В системе-источнике продукты не выделены в отдельную сущность. Меню в системе-источнике — это поле в документе, представляющем ресторан. 
Добавьте в DAG новый шаг для заполнения таблицы dds.dm_products, который:
Парсит JSON-документ, представляющий заказ, и извлекает из него продукты.
Проставляет в таблице dds.dm_products ссылку на запись в таблице dds.dm_restaurants.
Как заполнять таблицу:
id — автогенерируемое значение, int. Пример: 1.
product_id — id продукта из системы-источника, varchar. Пример: 6276e8cd0cf48b4cded0086c.
product_name — наименование продукта из системы-источника, text. Пример: ГРИЛАТА ОВОЩНАЯ ПО-МЕКСИКАНСКИ.
product_price — цена продукта из системы-источника, numeric. Пример: 60.00000.
active_from — update_ts из системы-источника, timestamp. Пример: 2022-10-31 18:00:14.000.
active_to — срок действия записи, timestamp. Если запись актуальна, то 2099-12-31. Пример: 2099-12-31 00:00:00.000.
restaurant_id — id версии ресторана (FK на dm_restaurants.id), int. Пример: 1.

Подсказка:
В STG данные о заказе лежат в формате JSON. Поэтому логика шага может выглядеть таким образом:
Прочитать данные из таблицы в STG.
Преобразовать JSON к объекту.
Найти версию ресторана по id и update_ts.
Вставить объекты с учётом историчности SCD2.
