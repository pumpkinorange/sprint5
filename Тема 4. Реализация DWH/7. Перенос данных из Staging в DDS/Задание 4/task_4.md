Задание 4. Заполните таблицу dm_timestamps
Значения для таблицы dds.dm_timestamps необходимо брать из таблицы заказов. В качестве основной даты возьмите дату получения заказом одного из финальных статусов — CLOSED или CANCELLED. Эта дата записана в поле date в JSON-структуре заказа.
Добавьте в DAG новый шаг для заполнения таблицы dds.dm_timestamps.
Как заполнять поля таблицы:
id — автогенерируемое значение, int. Пример: 1.
ts — дата и время создания заказа, timestamp. Пример: 2022-10-23 18:10:18.000.
year — год создания заказа, int. Пример: 2022.
month — месяц создания заказа, int. Пример: 10.
day — день создания заказа, int. Пример: 23.
date — дата создания заказа, date. Пример: 2022-10-23.
time — время создания заказа по UTC, time. Пример: 18:10:18.

Подсказка:
В STG данные о заказе лежат в формате JSON. Дата представлена в строковой форме. Вам необходимо распарсить строку, привести её к типу datetime, а затем извлечь параметры, фигурирующие в таблице в качестве отдельных столбцов.
Обратите внимание, что у отдельно взятого заказа может быть только один из указанных статусов — либо CLOSED, либо CANCELLED.
