Задание 6. Заполните таблицу dm_orders
Таблица заказов dds.dm_orders ссылается сразу на три удалённых измерения: «Рестораны», «Пользователи», «Временные метки». Загрузка должна учитывать эти ссылки. 
Добавьте в DAG новый шаг для заполнения таблицы dds.dm_orders, который: 
Прочитает данные о заказе.
Найдёт нужные версии объектов с учётом id связанных объектов и отметки времени заказа.
Вставит запись в заказ.
Как заполнять таблицу:
id — автогенерируемое значение, int. Пример: 1.
order_key — id заказа из системы-источника, varchar. Пример: 6355825bf9ab5d7232f00128.
order_status — статус заказа из системы-источника, varchar. Пример: CLOSED.
restaurant_id — id версии ресторана (FK на dm_restaurants.id), int. Пример: 1.
timestamp_id — id таймстемпа (FK на dm_timestamps.id), int. Пример: 1.
user_id — id пользователя (FK на dm_users.id), int. Пример: 1.

Подсказка:
В STG данные о заказе лежат в формате JSON. Поэтому логика шага может выглядеть таким образом:
Прочитать данные из таблицы в STG.
Преобразовать JSON к объекту.
Найти версию ресторана по id и update_ts.
Найти версию пользователя по id и update_ts.
Найти временную метку по date в заказе.
Вставить объект заказа в таблицу.
