Теперь нужно загрузить данные из таблицы outbox.
Таблица outbox — буфер изменений в базе-источнике. В неё постоянно дописываются новые данные, а старые данные вычищаются. При очередном запуске необходимо вычитать все имеющиеся в таблице строки.
Чтобы доставить данные из таблицы outbox в систему-приёмник, на источнике заводится периодический процесс, читающий таблицу outbox и отправляющий данные из неё в систему-приёмник. После отправки сообщения данные из таблицы outbox удаляются.
Чтобы не скачивать повторно одни и те же данные из таблицы outbox при повторяющихся запусках DAG, нужна таблица, в которой будет сохраняться текущий прогресс. 
У вас уже заведена таблица для хранения текущего курсора загруженных данных, она называется stg.srv_wf_settings. Таблица состоит из таких полей:
id — суррогатный id записи, первичный ключ, автоинкремент.
workflow_key — ключ задачи, которая сохраняет свою отметку о загрузке данных. Таких ключей может быть несколько.
workflow_settings — произвольный набор параметров. Его структура может быть уникальной для каждой задачи, поэтому используется тип данных JSON.
В качестве курсора, то есть показателя прогресса заливки данных, будем использовать id последней сохранённой записи. Работать эта схема будет только если сохранять данные в PostgreSQL, то есть в хранилище данных, в порядке увеличения id.

Вычитали записи из таблицы outbox, в которых id больше, чем сохранённый, то есть вычитанный на первом шаге.
Сохранили данные в таблицу stg.bonussystem_events.
Сохранили последний записанный id в таблицу stg.srv_wf_settings.
