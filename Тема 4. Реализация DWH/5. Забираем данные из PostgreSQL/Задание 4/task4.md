Задание 4. Создайте DAG, реализующий перекладывание outbox
Дополните ранее созданный DAG ещё одной операцией — events_load.
Эта операция должна считывать текущее состояние загрузки — в таблице outbox есть автогенерируемый непрерывно возрастающий id записи. В качестве метки состояния рекомендуем сохранять именно этот id.
Логика операции может быть примерно следующей:
Считали из stg.srv_wf_settings состояние с предыдущего запуска — максимальный id, загруженный в прошлый раз.
Вычитали записи из таблицы outbox, в которых id больше, чем сохранённый, то есть вычитанный на первом шаге.
Сохранили данные в таблицу stg.bonussystem_events.
Сохранили последний записанный id в таблицу stg.srv_wf_settings.

⚙ Чтобы не загружать данные повторно и не делать дубликаты, этапы 3 и 4 нужно выполнять в одной транзакции.
